🧩 多模态与AIGC (Multimodal AI)

宠物健康 AI 智能诊断系统 (Pet Doctor)

点击查看详情页 ↗

全栈式宠物医疗辅助小程序，融合视觉、音频与文本的多模态大模型技术。

Qwen2.5-VL RAG (Milvus) FastAPI 微信小程序
  • 项目背景：针对宠物医疗资源不均问题，开发基于微调大模型的智能问诊系统。
  • 核心技术：
    1. 多模态诊断：使用 Qwen2.5-VL 处理用户上传的患处图片，结合文本描述生成病理分析。
    2. 声音情感分析：集成音频模型识别宠物叫声情绪，辅助判断宠物状态。
    3. RAG 知识库：构建基于 Milvus 的兽医知识库，确保护理建议的专业性。
  • 架构设计：采用“后融合”策略，独立处理视觉流与音频流，最后由 LLM 整合生成综合诊断报告。
  • 成果：实现了口腔、皮肤等5大科室的初步筛查，并成功上线微信小程序端。

👁️ 计算机视觉 (Computer Vision)

实心球智能检测与测距系统

北大信息技术科创中心 | 算法实习项目

YOLOv8-改进 BiFPN C++ SAHI
  • 痛点解决：针对实心球目标小、运动模糊、背景干扰强的问题，提出了一套高精度检测方案。
  • 算法改进：引入 BiFPN 特征金字塔和轻量级 FRMHead，配合 NWD Loss 和 SAHI 切片推理技术，显著提升小目标检出率。
  • 落点与测距：设计了“轨迹聚类+趋势反转”算法精准定位落点；开发分区投影+偏差曲面矫正算法进行测距。
  • 工程落地：完成模型 ONNX 转 BMODEL 量化，使用 BMCV 进行端侧 C++ 部署，修复精度问题（统一 double 精度）。
  • 成果：小目标 mAP 达到 112%（相对基线），落地帧误差≤1帧，测距误差≤1cm，已撰写专利。

PMMW 违禁品轻量化检测

学术研究项目 | 论文/竞赛

模型压缩 剪枝量化 TensorRT
  • 任务：在被动毫米波（PMMW）图像中实时检测隐匿违禁品。
  • 模型优化：Backbone 融合深度可分离卷积，Neck 去除冗余特征层；检测头采用多支解耦与非对称压缩。
  • 部署策略：设计自适应校准算法，实施 FP32-INT8 全量化。
  • 成果：模型压缩率达 26.98%，推理速度提升 37.5%，低对比度场景下 mAP 提升 4.2%。

复杂场景烟火检测与量化

北大信息技术科创中心 | 算法实习项目

数据清洗 QAT量化 工程落地
  • 难点：聚集场景下误检率高，且需并在端侧设备运行。
  • 策略：重建数据集（加硬负样本），设计双版本模型策略：
    1. 低召回高精度版（烟火合一 + 连续帧触发）：用于极低误报场景。
    2. 高召回低精度版（仅火）：用于全检出场景。
  • 量化：从 DeepSparse QAT 转为“FP预训→QAT微调→校准”流程，并对齐算子，成功在客户侧交付。

📈 搜索与推荐系统 (Search & RecSys)

OPE：面向多任务学习的精排优化模型

学术研究项目 | 推荐算法

多任务学习 (MTL) MMOE/PLE Feature Selection
  • 背景：解决 MMOE/PLE 等模型在离线训练时的任务冲突与“负迁移”问题。
  • 创新点 (OPE结构)：
    1. 特征优选：每个任务独立选择最相关特征子集，物理隔离噪声。
    2. 个性化 Embedding：共享特征ID但使用独立 Embedding 参数，实现任务定制化表达。
  • 成果：在公开广告数据集上，CTR任务 AUC 提升 0.10%，CVR 提升 0.15%，LogLoss 下降约 0.5%。

基于元学习的冷启动 Embedding 框架

学术研究项目 | 冷启动优化

Meta Learning (MAML) Contrastive Learning
  • 目标：利用元学习（Meta Learning）解决新物品（Item）的 ID Embedding 初始化问题。
  • 方法：构造 ID Embedding 生成器，使用 MAML 训练；引入对比学习增强任务，拉近同源 Embedding 距离，推远异源距离；对交互序列去噪。
  • 成果：离线 GAUC 从 0.85 提升至 0.86。

🎵 音频算法 (Audio Algorithms)

短时脉冲音检测与异常监控

巅峰表现 | 音频算法实习

Log-Mel/MFCC TC-ResNet 异常检测
  • 难点：脉冲音极短、易被背景噪掩盖，且突变特征弱。
  • 算法：
    1. 数据增强：通过异常样本扩充与首/中/尾位置混入，纠正分布偏置。
    2. 模型构建：采用 Log-Mel/MFCC 特征 + 时间切块，构建 TC-ResNet/DS-CNN 监督基线及 MAE 自监督基线。
    3. 异常检测：构建向量库，使用 TSNE 抽样 + P95/P99 阈值冷却机制进行异常判定。
  • 全链路工程：完成从前端数据配置到训练、测试的自动化脚本（YAML/JSON配置，BPMN工作流）。
